# 05 — Handoff para constructor de landing pages

## Objetivo
Construir una landing **rápida, limpia y muy clara**, basada en el sistema visual adjunto y en el copy aprobado.

## Herramienta sugerida
Puede construirse en:
- Framer
- Webflow
- WordPress + Elementor
- Builder propio

## Requisitos técnicos mínimos
- responsive real,
- sticky header,
- CTA a WhatsApp visible en desktop y móvil,
- scroll suave por anclas,
- formulario corto,
- carga ligera,
- SEO básico editable,
- integración con Google Analytics / Meta Pixel / Tag Manager si luego se requiere.

## Estructura recomendada de navegación
- Inicio
- Servicios
- Cómo funciona
- FAQ
- Contacto

## Orden exacto a maquetar
1. Header
2. Hero
3. Barra de confianza
4. Servicios
5. Cómo funciona
6. Diferenciales
7. Casos frecuentes
8. FAQ
9. Contacto
10. Footer

## Prioridades de maquetación
### Alta prioridad
- claridad del hero,
- calidad del H1,
- contraste de botones,
- scroll visual limpio,
- CTA principal siempre visible,
- formulario corto.

### Media prioridad
- microanimaciones,
- transiciones,
- hover elegante,
- gradientes suaves.

### Baja prioridad
- ilustraciones complejas,
- efectos visuales pesados,
- sliders innecesarios,
- carruseles.

## Comportamientos UX
- primer CTA visible sin scroll,
- el usuario nunca debe preguntarse “qué hacen”,
- cada sección debe cerrar con CTA o enlace contextual,
- en móvil, el botón de WhatsApp puede fijarse abajo como barra flotante discreta.

## Reglas de contenido
- no agregar lorem ipsum,
- no reescribir el copy a estilo corporativo pesado,
- no meter jerga jurídica en subtítulos,
- no inventar testimonios ni casos,
- si no existen fotos reales, usar composición tipográfica + logo + shapes.

## Reglas de conversión
- un CTA principal dominante: WhatsApp,
- CTA secundarios subordinados,
- formularios con 5 campos máximo,
- no obligar al usuario a navegar por muchas páginas.

## Reglas de marca
- logo con espacio suficiente,
- violeta sobrio como acento,
- tipografía moderna,
- fondo limpio,
- estética consistente entre mobile y desktop.

## Entregable esperado del constructor
- landing publicada,
- editable,
- con textos cargados,
- con formulario funcional,
- con enlaces legales listos,
- optimizada para móvil.
